# План приложения "Тест-викторина"

## 1. Концепция

Простое SPA-приложение на чистом HTML/CSS/JS. Пользователь проходит тест из 7 вопросов, выбирая варианты ответов. В конце получает результат на основе суммы баллов.

**Технологии:** HTML5 + CSS3 + Vanilla JS (ES6+)
**Запуск:** открыть `index.html` в браузере
**Хранение:** localStorage (сохранение последнего результата)

## 3. Компоненты и их ответственность

### 3.1. `questions.js` — данные вопросов

Экспортирует массив объектов:

```js
export const questions = [
  {
    id: 1,
    text: "Прежде всего, Вы бы описали себя как...",
    answers: [
      { id: "1a", text: "Человека среднего роста", score: { ... } },
      { id: "1b", text: "Человека высокого роста", score: { ... } },
      { id: "1c", text: "Человека невысокого роста", score: { ... } },
    ]
  },
  // ... ещё 6 вопросов
];
```

Каждый ответ содержит `score` — объект с баллами по категориям результатов (ключи — ID результата, значения — число баллов). Это позволит гибко настраивать подсчёт, когда придут варианты результатов.

### 3.2. `results.js` — типы результатов и подсчёт

- Определяет типы результатов (например, `warrior`, `mage`, `rogue` и т.д.)
- Содержит функцию `calculateResult(answers)`, которая суммирует баллы по всем выбранным ответам и возвращает ID результата с наибольшим количеством баллов
- Содержит объект с описаниями результатов: заголовок, текст, изображение/иконка

Пока будет заглушка с 3-4 демо-результатами. Когда пользователь пришлёт варианты — заменим.

### 3.3. `app.js` — ядро приложения

- Управление состоянием: текущий вопрос, выбранные ответы (`Map<questionId, answerId>`)
- Рендеринг экранов: вопрос → результат
- Обработчики событий: выбор ответа, навигация (назад/далее), рестарт
- Сохранение/загрузка из localStorage

### 3.4. `index.html` — каркас

- Подключение скриптов (type="module")
- Контейнер `<div id="app">` для динамической отрисовки
- Мета-теги, заголовок

### 3.5. `style.css` — стили

- Минималистичный, аккуратный дизайн
- Адаптивность под мобильные
- Плавные переходы между вопросами
- Единая цветовая схема

## 4. Поток приложения

```
[Стартовый экран] → [Вопрос 1] → [Вопрос 2] → ... → [Вопрос 7] → [Результат]
                                                                       ↓
                                                              [Кнопка "Пройти заново"]
```

- На стартовом экране: название теста, краткое описание, кнопка "Начать"
- На каждом вопросе: номер вопроса, текст, варианты ответов (radio-style), кнопки "Назад" / "Далее"
- На последнем вопросе кнопка "Завершить" вместо "Далее"
- На экране результата: заголовок, описание, иконка, кнопка "Пройти заново"

## 5. Детали реализации

### Состояние (app.js)

```js
const state = {
  currentQuestionIndex: 0,
  answers: new Map(), // questionId → answerId
  isFinished: false,
};
```

### Рендеринг

- Функция `render()` — читает `state`, определяет какой экран показать, вызывает соответствующий рендерер
- `renderStartScreen()` — стартовый экран
- `renderQuestion(index)` — вопрос с вариантами
- `renderResultScreen()` — результат

### Подсчёт результатов

```js
function calculateResult(answers) {
  const scores = {};
  for (const [questionId, answerId] of answers) {
    const question = questions.find(q => q.id === questionId);
    const answer = question.answers.find(a => a.id === answerId);
    for (const [resultId, points] of Object.entries(answer.score)) {
      scores[resultId] = (scores[resultId] || 0) + points;
    }
  }
  // Возвращаем resultId с максимальным количеством баллов
  return Object.entries(scores).sort((a, b) => b[1] - a[1])[0][0];
}
```

1. Обновить `results.js` — добавить типы результатов, их описания и распределение баллов
2. Обновить `questions.js` — проставить `score` для каждого ответа
3. Возможно, обновить `style.css` — добавить иконки/цвета для результатов
